# 如何查看调试信息

应用已集成日志系统，所有调试信息自动记录到日志文件。

## 📁 日志文件位置

Windows: `C:\Users\你的用户名\.file_finder_logs\app_YYYYMMDD.log`

例如：`C:\Users\minxin.han\.file_finder_logs\app_20260130.log`

## 🔍 查看方式

### 方法1：直接打开日志文件（推荐）

1. 按 `Win + R`，输入：
   ```
   %USERPROFILE%\.file_finder_logs
   ```

2. 打开最新的日志文件（按日期命名）

3. 日志内容示例：
   ```
   2026-01-30 13:45:23 [INFO] 配置文件版本: v2
   2026-01-30 13:45:24 [INFO] 已保存搜索状态: keywords="SOL 101"
   2026-01-30 13:45:25 [WARNING] 配置文件为空，使用默认配置
   2026-01-30 13:45:26 [ERROR] 配置文件JSON格式错误: ...
   ```

### 方法2：运行调试版exe

1. 运行 `编译_调试版.bat` 编译调试版本
2. 双击运行 `dist\FileFinder_Debug.exe`
3. 会弹出黑色控制台窗口，实时显示日志

### 方法3：命令行运行普通版

```cmd
cd dist
FileFinder.exe
```

控制台会显示日志输出（但窗口会在关闭应用后消失）

## 📊 日志级别说明

- **[INFO]** - 普通信息（配置加载、保存等）
- **[WARNING]** - 警告信息（配置为空等）
- **[ERROR]** - 错误信息（JSON解析失败、文件损坏等）

## 🐛 故障排查

如果遇到问题：

1. 查看最新的日志文件
2. 查找 `[ERROR]` 或 `[WARNING]` 级别的消息
3. 关注配置迁移相关的日志：
   - "配置文件版本: v1" - 旧版本
   - "检测到旧版本配置，开始迁移..." - 正在迁移
   - "配置迁移成功！" - 迁移完成

## 📝 日志保留

- 日志按天分割（每天一个文件）
- 不会自动清理，可手动删除旧日志
- 每个日志文件通常很小（几KB）

## ⚙️ 编译版本对比

| 特性 | 普通版 (FileFinder.exe) | 调试版 (FileFinder_Debug.exe) |
|------|------------------------|-------------------------------|
| 控制台窗口 | 无 | 有 |
| 日志文件 | ✓ | ✓ |
| 实时输出 | ✗ | ✓ |
| 适用场景 | 日常使用 | 问题排查 |
| 编译脚本 | 编译.bat | 编译_调试版.bat |
